# Orders overview

IPProxy exposes order APIs directly. Use dynamic orders for traffic-based rotating proxy resources and static orders for fixed IP resources.

## Endpoint summary

| Method | Endpoint | Description |
| --- | --- | --- |
| `POST` | `/open/app/order/create/v2` | Create a dynamic proxy order. |
| `POST` | `/open/app/static/order/create/v2` | Create a static proxy order. |
| `GET` | `/open/app/static/order/list/v2` | List static orders and proxy instances. |
| `GET` | `/open/app/static/order/statistics` | Query static order status counts. |
| `POST` | `/open/app/static/order/renew` | Renew a static proxy order or instance. |
| `POST` | `/open/app/static/order/{order_no}/cancel` | Release/cancel a static order. |

## Shared order fields

| Field | Description |
| --- | --- |
| `orderType` | `dynamic_proxy` or `static_proxy`. |
| `poolId` | Product number from product query. |
| `userId` | Target user ID. |
| `unitPrice` | Unit price. |
| `totalAmount` | Total order amount. |
| `remark` | Optional order note. |

## Status and response

Successful order creation returns `code: 0` and an order object in `data`. Important response fields include:

| Field | Description |
| --- | --- |
| `order_no` | IPProxy order number. |
| `app_order_no` | Application order number. |
| `status` | Current order status. |
| `product_no` / `pool_type` | Purchased product. |
| `amount` / `total_amount` | Charged amount. |
| `created_at`, `updated_at` | Order timestamps. |

Use [Static orders](/sessions/static) for fixed IP resources and [Dynamic orders](/sessions/dynamic) for traffic-based resources.
