# Best practices

## Authentication

- Keep username, password, and Bearer tokens server-side.
- Refresh the token by logging in again when the API returns `401`.
- Do not embed tokens in fingerprint browser scripts that are visible to end users.

## Order creation

- Query products immediately before creating orders so `inventory`, `duration`, and price are current.
- Use `product_no` as `poolId`.
- Check balance before create and renew calls.
- For agents, ensure `userId` belongs to the current agent.
- After a create call fails or times out, query the order list before retrying to avoid duplicate orders.

## Static proxy usage

- Query `/open/app/static/order/list/v2` after creation to read final instance credentials.
- Search by order number or IP when reconciling a user report.
- Release resources only after confirming they are no longer in use.
- For shared static products, pass the intended `projectId` and verify project inventory.

## Operational checks

- Keep an audit trail of request payloads, response `order_no`, and balance before/after order creation.
- Use `order_no` filters when reconciling transactions.
- Page through list APIs with `page` and `pageSize` / `page_size`; do not request more than 100 rows per page.
