# Telemetry

The current IPProxy public API documentation does not define client-side telemetry submission endpoints. For operational visibility, use order list, dashboard, resource stats, and transaction APIs.

Relevant endpoints:

| Endpoint | Purpose |
| --- | --- |
| `GET /open/app/dashboard/info/v2` | Account and resource summary. |
| `GET /open/app/static/order/list/v2` | Static order and proxy instance status. |
| `GET /open/app/static/order/statistics` | Static order status counts. |
| `GET /open/app/user/transactions` | Consumption records. |

If you need custom callback or telemetry ingestion, coordinate the schema with IPProxy support before implementation.
