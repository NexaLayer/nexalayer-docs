# 账号

ZTProxy 账号对应平台用户。代理商可以通过 API 创建并管理自己的下级用户。

## 创建用户

```bash
curl -X POST "https://backend.ztproxy.net/api/open/app/user/v2" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "sub_user_001",
    "password": "password123",
    "email": "user@example.com",
    "phone": "13812345678",
    "remark": "Created from API",
    "balance": 0
  }'
```

## 请求字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | :---: | --- |
| `username` | string | 是 | 3-50 个字符，必须唯一。 |
| `password` | string | 是 | 6-50 个字符。 |
| `email` | string | 否 | 邮箱地址。 |
| `phone` | string | 否 | 11 位手机号。 |
| `remark` | string | 否 | 备注。 |
| `balance` | number | 否 | 初始余额。 |

## 当前余额

```bash
curl -X GET "https://backend.ztproxy.net/api/open/app/user/balance/current" \
  -H "Authorization: Bearer <token>"
```

## 查询下级账号列表和余额

代理商可查询自己的直属下级普通用户，响应会返回每个子账号的余额和基础资料。

```bash
curl -X GET "https://backend.ztproxy.net/api/open/app/agent/sub-users?page=1&pageSize=20" \
  -H "Authorization: Bearer <token>"
```

可选查询参数：

| 参数 | 说明 |
| --- | --- |
| `page` | 页码，默认 `1`。 |
| `pageSize` | 每页数量，默认 `20`，最大 `100`。 |
| `search` | 按用户名、邮箱或手机号模糊搜索。 |
| `status` | 按账号状态筛选：`active`、`disabled`、`deleted`。 |
| `includeDeleted` | 是否包含已删除账号，默认 `false`。 |

## 代理商管理下级账号

以下接口仅供代理商账号操作自己的直属下级用户。目标用户必须满足：

- 系统校验目标账号属于当前登录代理商。
- 不是当前代理商自己。
- 是当前代理商名下的直属下级普通用户。
- 已删除账号不能再次编辑、调余额或切换状态。

### 调整下级用户余额

```bash
curl -X POST "https://backend.ztproxy.net/api/open/app/agent/sub-users/1001/balance" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 100,
    "remark": "API balance adjustment"
  }'
```

`amount` 为正数表示增加余额，为负数表示扣减余额。扣减后余额不能小于 0。

### 编辑下级用户账号

```bash
curl -X PATCH "https://backend.ztproxy.net/api/open/app/agent/sub-users/1001/account" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "sub_user_002",
    "password": "newPassword123"
  }'
```

`username` 和 `password` 至少传一个。此接口修改 ZTProxy 登录账号信息，不修改供应商侧账号字段。

### 启用或禁用下级用户

```bash
curl -X PATCH "https://backend.ztproxy.net/api/open/app/agent/sub-users/1001/status" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "disabled"
  }'
```

`status` 可选 `active` 或 `disabled`。

### 删除下级用户

```bash
curl -X DELETE "https://backend.ztproxy.net/api/open/app/agent/sub-users/1001" \
  -H "Authorization: Bearer <token>"
```

删除为软删除，账号状态会变为 `deleted`。此接口只用于删除直属下级普通用户。

## 仪表盘概览

```bash
curl -X GET "https://backend.ztproxy.net/api/open/app/dashboard/info/v2" \
  -H "Authorization: Bearer <token>"
```

该接口返回当前账号的仪表盘概览。
