# 架构

ZTProxy 对外 API 由 ZTProxy 后端直接提供。当前公开契约中没有额外的会话网关层，调用方直接访问 ZTProxy 的订单、产品、用户、余额和交易记录接口。

```text
客户系统 / 合作方系统
        |
        | HTTPS + Bearer Token
        v
https://api.wanyuagent.com/api
        |
        v
ZTProxy 后端
  - 登录认证和用户权限
  - 产品库存和价格
  - 动态/静态订单创建
  - 静态代理实例
  - 余额和交易记录
  - 必要时对接底层资源服务
```

## 职责划分

| 模块 | ZTProxy API 职责 |
| --- | --- |
| 认证 | 登录和 JWT Bearer Token 校验。 |
| 用户 | 代理商创建下级用户和权限检查。 |
| 产品 | 产品、地区、库存、项目和 CIDR 数据。 |
| 订单 | 动态/静态订单创建、查询、续费和释放。 |
| 动态代理 | 动态子账户、流量查询和代理提取。 |
| 账务 | 余额、消费记录、充值/调整记录。 |
| 代理访问 | 静态实例凭证。 |

## 请求流程

1. 调用方使用 `POST /auth/login` 登录。
2. 调用方保存 `data.token`，并在后续请求中使用 `Authorization: Bearer <token>`。
3. 查询产品和余额。
4. 为有权限的 `userId` 创建订单。
5. 查询订单列表或详情，获取代理实例并核对状态。
6. 查询交易记录进行账务审计。

## 域名

普通服务端集成使用生产 API 域名：

