# Introduction

IPProxy provides HTTP APIs for partners and agent systems to manage proxy resources programmatically. The public API covers login, user management, product and region query, dynamic proxy orders, static proxy orders, renewals, whitelist operations, dashboard balance, and transaction records.

## Base URL

All examples in this documentation use the production API domain:

```text
https://api.wanyuagent.com/api
```

For fingerprint browser integrations, call the same IPProxy API domain from the browser automation environment: `api.wanyuagent.com`.

## Response format

Most IPProxy APIs return the following shape:

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

Some legacy endpoints use `msg` instead of `message`:

```json
{
  "code": 0,
  "msg": "success",
  "data": {}
}
```

Treat `code: 0` as success. Non-zero `code` or an HTTP 4xx/5xx status means the request failed.

## Core concepts

| Concept | Description |
| --- | --- |
| User | An IPProxy account. Agents can create and manage their sub-users. |
| Product | A proxy resource package, identified by `product_no` / `poolId`. |
| Dynamic order | Traffic-based rotating proxy order. |
| Static order | Fixed IP order with region, duration, IP count, and optional project / CIDR settings. |
| Instance | A concrete proxy IP/port/credential record attached to a static order. |
| Balance | Account available amount used for order creation and renewal. |

## Main API areas

- **Authentication**: `POST /auth/login`
- **Users**: create sub-users and query current balance
- **Products and regions**: query available proxy products and service areas
- **Orders**: create dynamic/static proxy orders, list static orders, renew or release resources
- **Proxy access**: whitelist management and extracted proxy credentials
- **Billing records**: balance, recharge/adjustment records, and consumption records
