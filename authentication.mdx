# Authentication

IPProxy public APIs use JWT Bearer authentication. First call the login endpoint with your IPProxy username and password, then send the returned token in the `Authorization` header.

## Login

<CodeGroup>

```bash Request
curl -X POST "https://api.wanyuagent.com/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "your_username",
    "password": "your_password"
  }'
```

```json Response
{
  "code": 0,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "user": {
      "id": 1001,
      "username": "your_username",
      "is_admin": false,
      "is_agent": true,
      "balance": 1000,
      "status": "active"
    }
  }
}
```

</CodeGroup>

## Use token

Send the token on every authenticated request:

```bash
Authorization: Bearer <token>
```

Example:

```bash
curl -X GET "https://api.wanyuagent.com/api/open/app/dashboard/info/v2" \
  -H "Authorization: Bearer <token>"
```

## Token errors

| HTTP | Message | Meaning |
| --- | --- | --- |
| 401 | 未提供认证令牌 | Missing `Authorization` header. |
| 401 | 无效的认证令牌 | Token is invalid or expired. |
| 401 | 用户不存在 | Token user no longer exists. |
| 403 | 账号未激活，请先前往邮箱点击激活链接 | Account exists but is not active. |

Store credentials and tokens in a secret manager or server-side environment variables. Do not expose them in frontend code or logs.
