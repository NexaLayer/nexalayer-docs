# Recharge

Recharge and payment flows are account-specific. Use the transaction APIs to audit recharge and adjustment records, and use the dashboard balance API to verify final balance.

## Related endpoints

| Method | Endpoint | Description |
| --- | --- | --- |
| `GET` | `/transaction/agent/transactions` | Recharge and adjustment records. |
| `GET` | `/open/app/user/transactions` | Consumption records. |
| `GET` | `/open/app/dashboard/info/v2` | Current balance and dashboard summary. |
| `GET` | `/open/app/user/balance/current` | Current authenticated user balance. |

## Recommended verification flow

1. Query current balance.
2. Complete the recharge or balance adjustment through the agreed payment process.
3. Query `/transaction/agent/transactions` to confirm the record.
4. Query balance again to confirm the updated amount.

For online payment integrations, contact IPProxy support to confirm which payment endpoint is enabled for your account and environment.
