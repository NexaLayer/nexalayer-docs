# 计费与余额

IPProxy 后端 CNY 余额仍是唯一资金事实源。NexaLayer 使用 IPProxy 当前有效 USD/CNY 汇率，将余额、产品价格和交易流水换算为 USD 展示，不维护第二套可消费余额。

所有对外 USD 余额、单价和扣费金额统一保留两位小数。汇率保留四位；充值订单返回的准确 USDT 支付金额用于识别转账，不得自行四舍五入。

当前计费规则：

| 项目 | 价格 |
| --- | --- |
| 试用 | 联系 Telegram 售后 @ztproxy 配置资源 |
| 最低充值 | $10，USDT TRC20 自助充值 |
| Dynamic Session 资源 | 以 `/products` 实时 USD 换算价格为准 |
| Static Session 资源 | 以 `/products` 实时 USD 换算价格为准 |
| Session 管理 | 暂时免费 |

## 查询价格

```bash
curl "https://api.nexalayer.net/v1/billing/pricing" \
  -H "X-API-Key: $NEXALAYER_API_KEY"
```

## 查询余额

```bash
curl "https://api.nexalayer.net/v1/billing/balance" \
  -H "X-API-Key: $NEXALAYER_API_KEY"
```

## 创建充值订单

```bash
curl -X POST "https://api.nexalayer.net/v1/billing/recharge" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: $NEXALAYER_API_KEY" \
  -H "Idempotency-Key: recharge-001" \
  -d '{ "amount_usd": 10, "currency": "USDT" }'
```

低于 $10 的充值请求会被拒绝。响应包含二维码所需的准确 `amount_usdt`、`wallet_address`、`network` 和订单汇率。

## 查询交易记录

```bash
curl "https://api.nexalayer.net/v1/billing/transactions" \
  -H "X-API-Key: $NEXALAYER_API_KEY"
```

## 接入建议

- 创建付费 Session 前先读取余额
- 余额和实际资源扣款均由后端负责，NexaLayer 不执行本地预扣
- 写操作携带 `Idempotency-Key`
- 售后配置资源后，创建 Session 时使用 `provisioning_mode=existing`，避免重复下单或扣费
- 不要把充值、扣费和清理动作混在一个不可恢复的脚本里
- 如需试用，请联系 https://t.me/ZTPROXY（@ztproxy）；也可以自行充值后直接使用
