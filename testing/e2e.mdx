# E2E tests

A complete end-to-end test should cover the real integration path:

1. Health check: `GET https://api.wanyuagent.com/health`.
2. Login: `POST /api/auth/login`.
3. Dashboard: `GET /api/open/app/dashboard/info/v2`.
4. Products: `POST /api/open/app/product/query/v2`.
5. Areas: `GET /api/open/app/product/area/v2`.
6. Balance: `GET /api/open/app/user/balance/current`.
7. Create order: static or dynamic, using the smallest available test product.
8. Query order: static order list or dynamic order response.
9. Transactions: `GET /api/open/app/user/transactions`.
10. Cleanup: release static test resources where applicable.

Security assertions:

- Protected endpoints reject missing tokens with `401`.
- Invalid tokens are rejected.
- Agent accounts cannot operate on unrelated users.
- Invalid products, durations, or insufficient balance return clear failure responses.
