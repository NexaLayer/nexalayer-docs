# Integration tests

Integration tests should verify that API calls produce the expected IPProxy records.

## Static order flow

1. Login.
2. Query products and choose a product with sufficient inventory.
3. Query balance.
4. Create a static order with `POST /api/open/app/static/order/create/v2`.
5. Save `order_no`.
6. Query `/api/open/app/static/order/list/v2?search=<order_no>`.
7. Assert the order exists and proxy instances are returned when available.
8. Query `/api/open/app/user/transactions?order_no=<order_no>`.
9. Verify consumption and balance changes.

## Dynamic order flow

1. Login.
2. Query dynamic product data.
3. Create a dynamic order with `POST /api/open/app/order/create/v2`.
4. Assert the returned order has `order_no`, `pool_type`, amount, and active status.
5. Query transaction records for the order.

## Permission flow

Use an agent account and verify:

- The agent can create orders for its own sub-users.
- The agent cannot create orders for unrelated users.
- Normal users cannot query other users' transaction records.
