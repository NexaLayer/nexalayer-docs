# Testing overview

Use a staged test flow that exercises the same endpoints your production integration will call.

## Recommended layers

| Layer | Scope |
| --- | --- |
| Smoke | Login, dashboard, product query, and static order list. |
| Order flow | Create a small dynamic or static order, then query order status. |
| Permission | Confirm agents can only operate on permitted sub-users. |
| Billing | Compare balance and transaction records before and after an order. |
| Error handling | Missing token, invalid token, insufficient balance, invalid product. |

## Environment

- Base URL: `https://api.wanyuagent.com/api`
- Health URL: `https://api.wanyuagent.com/health`
- Use a dedicated test account and test sub-user.
- Avoid production-sized orders during automated tests.

## Cleanup

For static proxy tests, release test resources when finished where the product policy allows it.
