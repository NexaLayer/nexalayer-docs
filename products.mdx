# Products

Products define which proxy resource can be purchased. Use product query APIs before creating orders so you can pick the correct `poolId`, region, duration, IP count, and price.

## Product query

```bash
curl -X POST "https://api.wanyuagent.com/api/open/app/product/query/v2" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "proxyType": 103
  }'
```

Useful response fields:

| Field | Description |
| --- | --- |
| `product_no` | Product number. Use as `poolId` when creating orders. |
| `product_name` | Display name. |
| `proxy_type` | Proxy category. Common values: `103` static, `104` dynamic. |
| `inventory` | Available quantity. |
| `country_code`, `state_code`, `city_code` | Location fields. |
| `duration`, `unit` | Supported duration configuration. |
| `global_price` | Default sales price where configured. |
| `band_width`, `band_width_price` | Bandwidth and bandwidth pricing where applicable. |
| `cidr_info` | CIDR-level inventory and pricing where applicable. |
| `project_list` | Project inventory for shared static proxy products. |

## Region query

```bash
curl -X GET "https://api.wanyuagent.com/api/open/app/product/area/v2?proxyType=103&productNo=ZT_US_New%20York%20City_2" \
  -H "Authorization: Bearer <token>"
```

Use the returned region/country/city codes as `regionCode`, `countryCode`, and `cityCode` in static order requests.

## Common product types

| Product | Typical use | Order endpoint |
| --- | --- | --- |
| Static proxy | Fixed IP, fixed duration, stable proxy credentials | `POST /open/app/static/order/create/v2` |
| Dynamic proxy | Traffic-based rotating proxy resources | `POST /open/app/order/create/v2` |
| Shared static proxy | Static product with `project_list` / project inventory | `POST /open/app/static/order/create/v2` with `projectId` |

## Selecting a product

1. Query products by `proxyType`.
2. Check `inventory` and region fields.
3. Confirm duration rules. For normal static products, `unit` supports `1` day, `3` month, and `4` year; valid durations depend on product configuration.
4. Use `product_no` as `poolId` when creating the order.
5. If the product has `project_list`, choose a `projectId` with sufficient inventory.
